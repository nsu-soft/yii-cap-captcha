## 📋 Обзор

**yii-cap-captcha** — это PHP-библиотека для фреймворка **Yii2**, предоставляющая надежный и удобный интерфейс для взаимодействия с автономным сервером **[Cap Captcha](https://github.com/tiagozip/cap)**. Пакет обеспечивает бесшовную интеграцию защиты от ботов в ваши приложения на Yii2, используя внешний сервис для генерации капчи, доставки заданий и проверки ответов.

Данный компонент следует паттернам проектирования Yii2 и полностью совместим со стандартами PSR-7 (HTTP-сообщения) и PSR-18 (HTTP-клиент).

---

## ✨ Возможности

- 🔐 **Полная интеграция с Cap Captcha API**: Доступ ко всем эндпоинтам, включая получение заданий (challenge), обмен на токен доступа (redeem), проверку токена (siteverify) и операции по администрированию.
- 🧩 **Архитектура компонента Yii2**: Готовый к использованию компонент с поддержкой стандартной конфигурации приложения Yii.
- 🔑 **Комплексное управление ключами**: Программное добавление, просмотр, обновление, ротация и удаление ключей сайта и API-ключей.
- 🐳 **Поддержка Docker**: Предварительно настроенный `docker-compose.yml` для мгновенного развертывания сред локальной разработки и тестирования.
- 🧪 **Тестирование с Codeception**: Полный набор тестов, охватывающий модульные (Unit) и функциональные (Functional) сценарии.
- 📦 **Соответствие PSR**: Построено на интерфейсах PSR-7 (HTTP-сообщения) и PSR-18 (HTTP-клиент) для максимальной совместимости.
- 🔄 **Гибкий HTTP-клиент**: Работает с любым PSR-18 совместимым клиентом через автообнаружение (discovery) или явную конфигурацию.

## 📦 Установка


Если у вас не установлен Composer, вы можете сделать это, [следуя инструкции](https://getcomposer.org/doc/00-intro.md).

Затем установите пакет с помощью Composer:

```bash
composer require nsu-soft/yii-cap-captcha
```

Также в вашем проекте должен быть установлен HTTP-клиент. Вы можете использовать стандартный [yiisoft/yii2-httpclient](https://packagist.org/packages/yiisoft/yii2-httpclient). Кроме того, необходимо установить реализацию PSR-17. Например:

```bash
composer require yiisoft/yii2-httpclient
composer require nyholm/psr7
```

Или вы можете использовать одну из [реализаций PSR-18](https://packagist.org/providers/psr/http-client-implementation).

Например:

```bash
composer require guzzlehttp/guzzle
```

Пакет будет использовать тот HTTP-клиент, который обнаружит в проекте. Для поиска клиента используется пакет [php-http/discovery](https://packagist.org/packages/php-http/discovery) .

## ⚙️ Конфигурация

### Шаг 1: Сгенерируйте ключи в административной панели сервера Cap Captcha

Сгенерируйте ключи доступа к серверу Cap Captcha, как описано на [официальном сайте](https://capjs.js.org/guide/) в разделе «Setting up your server» (Настройка вашего сервера).

### Шаг 2: Добавьте конфигурацию в ваше приложение Yii 2

Добавьте параметры вашего сервера Cap Captcha в конфигурацию вашего приложения:

```php
    'components' => [
        'captcha' => [
            'class' => NsuSoft\Captcha\Cap::class,

            'server' => 'http://localhost',   // Базовый URL сервера Cap Captcha
            'port' => 3000,                   // Порт сервера (по умолчанию: 3000)
            
            // Учетные данные сайта (предоставляются при регистрации сайта)
            'siteKey' => 'your-site-key',     // Публичный идентификатор сайта
            'secretKey' => 'your-secret-key', // Приватный ключ для проверки клиентских запросов

            // Опционально: API-ключ для операций администрирования
            'apiKey' => '',                   // Оставьте пустым, если не управляете ключами программно
        ],
    ],
```

## 💻 Использование

### Получение экземпляра компонента

Вы можете обратиться к настроенному компоненту из любого места вашего приложения Yii2:

```php
$cap = Yii::$app->captcha;
```

### Справочник методов

JSON-схемы находятся в директории `tests/Support/Data/Cap`.

| Метод | Описание | Параметры | Возвращаемые значения |
|-------|----------|-----------|------------|
| `siteVerify(string $response)` | Альтернативная конечная точка проверки (в стиле reCAPTCHA) | `$response`: токен из эндпоинта `/{siteKey}/redeem` | См. JSON-схему в `Main/siteverify.200.json` |
| `getAbout()` | Получает метаданные сервера | — | См. JSON-схему в `Server/about.200.json` |
| `logout(string $session)` | Завершает указанную сессию | `$session` | `null` |
| `getKeys()` | Получает все ключи сайтов | — | См. JSON-схему в `Server/Keys/index.200.json` |
| `createKey(string $name)` | Создает ключ сайта | `$name` | См. JSON-схему в `Server/Keys/post.200.json` |
| `viewKey(string $siteKey)` | Просматривает ключ сайта | `$siteKey` | См. JSON-схему в `Server/Keys/get.200.json` |
| `deleteKey(string $siteKey)` | Удаляет ключ сайта вместе с его секретом | `$siteKey` | См. JSON-схему в `Server/Keys/delete.200.json` |
| `configKey(string $siteKey, array $options = [])` | Настраивает ключ сайта | `$siteKey`, `$options = ['challengeCount' => int, 'difficulty' => int, 'name' => 'new-name', 'saltSize' => int]` | См. JSON-схему в `Server/Keys/config.200.json` |
| `rotateSecret(string $siteKey)` | Обновляет секретный ключ для указанного ключа сайта | — | См. JSON-схему в `Server/Keys/rotate-secret.200.json` |
| `getApiKeys()` | Получает все API-ключи | — | См. JSON-схему в `Server/Settings/apikeys.index.200.json` |
| `createApiKey(string $name)` | Создает API-ключ | `$name` | См. JSON-схему в `Server/Settings/apikeys.post.200.json` |
| `deleteApiKey(string $id)` | Удаляет API-ключ | `$id` | См. JSON-схему в `Server/Settings/apikeys.delete.200.json` |
| `deleteLastApiKey(string $name)` | Удаляет последний добавленный API-ключ по его имени | `$name` | См. JSON-схему в `Server/Settings/apikeys.delete.200.json` |
| `getSessions()` | Получает токены всех сессий | — | См. JSON-схему в `Server/Settings/sessions.200.json` |

### Пример: Проверка ответа пользователя

```php
use NsuSoft\Captcha\Exceptions\JsonDecodeException;
use Psr\Http\Client\ClientExceptionInterface;
use Yii;

$cap = Yii::$app->captcha;

try {
    $response = $cap->siteVerify($responseToken); // токен, полученный от @cap.js/widget
    
    if ($response->success) {
        // ✅ Валидация капчи прошла успешно
        Yii::info("Капча пройдена");
    }
} catch (ClientExceptionInterface $e) {
    // Ошибка сети или взаимодействия с сервером
    Yii::error("Сервис капчи недоступен: " . $e->getMessage());
} catch (JsonDecodeException $e) {
    // Ошибка декодирования JSON из ответа сервера
    Yii::error("Ошибка декодирования JSON: " . $e->getMessage());
}
```

### Управление ключами (требуется `apiKey`)

> ⚠️ Для работы этих методов в компоненте должен быть настроен корректный `apiKey`.

```php
use Yii;

$cap = Yii::$app->captcha;

// Список всех зарегистрированных ключей сайта
$keys = $cap->getKeys();

// Создание нового ключа сайта
$newKey = $cap->createKey('Мой рабочий сайт');

// Просмотр конфигурации конкретного ключа
$keyInfo = $cap->viewKey('your-site-key');

// Обновление конфигурации ключа
$cap->configKey('your-site-key', [
    'challengeCount' => 10,
    'difficulty' => 5,
    'name' => 'new-site-key-name',
    'saltSize' => 40,
]);

// Ротация секретного ключа (аннулирует старый секрет)
$cap->rotateSecret('your-site-key');

// Удаление ключа сайта (необратимо)
$cap->deleteKey('your-site-key');
```

### Управление API-ключами

> ⚠️ Для работы этих методов в компоненте должен быть настроен корректный `apiKey`.

```php
use Yii;

$cap = Yii::$app->captcha;

// Список всех API-ключей
$apiKeys = $cap->getApiKeys();

// Создание нового API-ключа с описанием
$cap->createApiKey('Инструмент CI/CD развертывания');

// Удаление API-ключа по его ID
$cap->deleteApiKey('ak_1234567890');

// Удаление последнего созданного ключа с указанным именем
$cap->deleteLastApiKey('Инструмент CI/CD развертывания');
```

### Управление сессиями

> ⚠️ Для работы этих методов в компоненте должен быть настроен корректный `apiKey`.

```php
use Yii;

$cap = Yii::$app->captcha;

// Получение списка активных сессий
$sessions = $cap->getSessions();

// Завершение конкретной сессии
$cap->logout('session-id-here');
```

## ⚠️ Обработка ошибок

Компонент выбрасывает исключения как для устранимых, так и для критических ошибок. Всегда оборачивайте вызовы методов в блоки `try-catch`.

### Типы исключений

| Исключение | Когда выбрасывается | Рекомендуемое действие |
|------------|---------------------|------------------------|
| `Psr\Http\Client\RequestExceptionInterface` | Исключение при неудачном запросе. | Запрос из этого пакета несовместим с сервером Cap Captcha. Создайте issue в этом репозитории. |
| `Psr\Http\Client\NetworkExceptionInterface` | Выбрасывается, когда запрос не может быть завершен из-за проблем с сетью. | Проверьте сетевое подключение. |
| `NsuSoft\Captcha\Exceptions\ResponseExceptionInterface` | Тело ответа с сервера Cap Captcha не является валидным JSON. | Несовместимая версия сервера Cap Captcha с данным пакетом. Создайте issue в этом репозитории. |

## 🔐 Безопасность

> ⚠️ **Критические рекомендации по безопасности**

1. **Никогда не фиксируйте секреты в системе контроля версий**  
   Исключите секретный ключ и API-ключ из Git:

   ```php
   'secretKey' => getenv('CAPTCHA_SECRET_KEY'),
   'apiKey' => getenv('CAPTCHA_API_KEY'),
   ```

2. **Используйте конфигурацию, специфичную для окружения**  
   Поддерживайте отдельные конфиги для сред `dev`, `staging` и `prod`.

3. **Регулярно обновляйте ключи**  
   Периодически используйте `rotateSecret()` и безопасно обновляйте конфигурацию.

4. **Ограничьте доступ к административным эндпоинтам**  
   Убедитесь, что маршруты, защищенные `apiKey`, доступны только из доверенных IP или внутренних сетей.

5. **Используйте HTTPS в продакшене**  
   Всегда используйте `https://` для URL `server` при развертывании продуктовой версии.

6. **Мониторьте и логируйте подозрительную активность**  
   Фиксируйте неудачные попытки валидации и необычные паттерны для аудита безопасности.
